# ADR-0003: `_temp/` внутри vault, а не в OS temp

- **Status:** accepted
- **Date:** 2026-04-11 *(ratified; решение в v2.0.0-alpha.1)*
- **Scope:** все скиллы с субагентами, спецификация формата JSON-отчётов

## Context

Субагенты пишут JSON-отчёты, которые потом читает главный контекст (см. [ADR-0002](0002-subagents-readonly-on-vault.md)). Нужно место для этих файлов со следующими ожиданиями:

- **Общий путь** для субагентов и главного контекста — без хитрой координации, без env-переменных
- **Переживает рестарт сессии**, но не обязано переживать чистку диска
- **Видимое пользователю** для ручной проверки при отладке, но **не захламляющее** основной граф vault (Obsidian не должен индексировать эти JSON как заметки)
- **Не требует** специальных прав или настройки OS
- **Кросс-платформенно**: одинаково работает на Windows, Linux, Mac

## Decision

**Директория `_temp/` в корне vault** с именованием `{skill}-{task}-{timestamp}.json`.

- Timestamp в формате `YYYY-MM-DDTHH-MM` (без двоеточий — Windows compat)
- Папка добавляется в `vault-config` excluded areas → Obsidian её не индексирует, не показывает в граф-view
- Папка добавляется в `.gitignore` vault-а (если vault под git)
- **Lifecycle:**
  - Создаётся субагентом через `Write` tool
  - Читается главным контекстом через `Read` (по секциям для больших файлов)
  - Удаляется после apply-фазы (skill спрашивает в `cautious`/`balanced`, удаляет автоматически в `auto`)
- **Stale detection:** файлы старше 24ч — предупреждение и предложение удалить на следующем запуске

## Alternatives considered

1. **OS temp (`/tmp` или `%TEMP%`).** Отвергнуто:
   - Разные пути на Windows/Linux/Mac — усложнение кода субагентов и wrapper-скриптов
   - Пользователю трудно туда заглянуть при отладке (особенно на Windows — `%TEMP%` скрыт)
   - Некоторые OS чистят `/tmp` при рестарте — потеря отчётов между итерациями одной задачи
2. **`.obsidian-brain-temp/` в корне vault (dotfile).** Отвергнуто: Obsidian по умолчанию скрывает dotfiles, но пользователи всё равно путаются. Видимая `_temp/` с префиксом `_` согласуется с существующей конвенцией vault (`_archive/`, `_templates/`, `_assets/`, `_inbox/`, `_moc/`).
3. **В папке плагина (`~/.claude/plugins/.../obsidian-brain/_temp/`).** Отвергнуто: плагин установлен глубоко в cache-директории, пользователь туда не смотрит. Пути субагентов становятся длинными и хрупкими при переустановке плагина.
4. **В git-репо плагина (`docs/temp/`).** Отвергнуто: смешивание runtime-данных с исходниками. При `git clean` теряется состояние между сессиями.

## Consequences

**Плюсы:**
- Один путь, кросс-платформенно, никаких env-переменных
- Пользователь видит отчёты в своём vault-е — может открыть в текстовом редакторе или даже в Obsidian (как JSON-preview)
- Именование с timestamp предотвращает конфликты при параллельных запусках одного скилла
- `_temp/` рядом с `_archive/` — визуально понятно, что это служебная папка

**Минусы:**
- **Засоряет vault**, если cleanup ломается. Stale-detection (24h) это смягчает, но не убирает риск.
- Требует от пользователя **добавить `_temp/` в vault-config** — шаг при первичной установке. Упущение этого шага приводит к тому, что Obsidian пытается индексировать JSON-файлы.
- Если пользователь держит vault под git, нужно также добавить `_temp/` в `.gitignore` vault-а — ещё один ручной шаг.
